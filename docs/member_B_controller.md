# 成员 B：试验流程与状态机模块

## 负责模块

试验状态机、计时器、按钮状态控制、系统消息触发。

## 主要文件

| 文件 | 说明 |
|---|---|
| `services/test_controller.py` | 状态机核心 |
| `ui/main_window.py` | 按钮状态与界面事件 |

## 需要实现的功能

- Idle -> Preparing。
- Preparing -> Ready。
- Ready -> Recording。
- Recording -> Complete。
- 停止升温回到 Idle。
- 记录阶段每秒增加计时。
- 关键事件写入系统消息。
- 完成未保存时禁止覆盖当前试验。

## 与其他成员的接口

| 成员 | 接口 |
|---|---|
| A | 当前试验对象由数据库模块创建 |
| C | 每次 tick 调用仿真器更新温度 |
| D | 提供 `record_samples` 和 `build_result()` |
| E | UI 根据 `state` 控制按钮 |

## 测试方法

启动程序后点击开始升温，观察是否自动进入就绪；点击开始记录和停止记录，观察状态和消息。

## 最终交付物

状态流转稳定，按钮可用状态正确，系统消息清晰。
